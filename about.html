Building VaultComics: My First Real Website

May 16, 2025 4 min read

Project Overview

VaultComics was my university project and honestly, my first attempt at building something that looked like a real website. It's a comic book e-commerce site that I built to learn the basics of HTML, CSS, and JavaScript. Looking back, it's pretty basic, but it was where I started understanding how websites actually work.

What I Built

  • Product Catalog: A grid of comic books with images and prices
  • Shopping Basket: Basic add-to-cart functionality
  • Checkout Process: Simple form to complete purchases
  • Responsive Design: Made it work on different screen sizes

What I Struggled With

Making JavaScript Actually Work

I spent way too much time trying to get the shopping cart to update properly. The concept of event listeners was completely new to me, and I kept wondering why clicking buttons didn't do anything. Eventually figured out that I needed to actually tell JavaScript what to listen for.

CSS Layouts

Getting the product grid to look right was harder than I expected. I didn't understand flexbox or grid at all, so I used a lot of floats and positioning that probably wasn't the best approach. But it worked!

Key Takeaways

  • Start Simple: You don't need to build the next Amazon on your first try
  • Debugging is Learning: Every error message taught me something new
  • Visual Design Matters: Making something look good is as important as making it work
  • Documentation Helps: Writing down what I was trying to build kept me focused

Portfolio Website: Learning to Build from Scratch

April 7, 2025 - Present 5 min read

Project Overview

After completing VaultComics as my university project, I realized I needed to learn how to properly build a website from scratch. VaultComics was my first real website, but it was more of a learning exercise. This portfolio website represents my journey into understanding proper web development practices, JavaScript functionality, linking systems, and creating something that actually represents my skills professionally.

What I Actually Learned

  • Proper JavaScript: Moving beyond basic scripts to actual interactive functionality
  • File Linking: Understanding how HTML, CSS, and JavaScript files connect and communicate
  • Responsive Design: Making sites that work on phones, tablets, and desktops
  • Project Structure: Organizing code in a way that makes sense and is maintainable
  • GitHub Integration: Learning version control and proper repository management

Key Features I Built

  • Navigation System: Working links between pages and smooth scrolling
  • Project Showcase: Organizing my work in a way that makes sense to others
  • Timeline Section: Showing my learning journey chronologically
  • Contact Form: Actually functional contact system
  • Responsive Layout: Looks good on any device

Real Challenges I Faced

Understanding File Relationships

Coming from VaultComics, I didn't really understand how HTML files link to CSS and JavaScript. I kept breaking links and wondering why styles weren't loading. I had to learn the difference between relative and absolute paths, and how browsers actually find and load resources.

Making JavaScript Actually Work

In VaultComics, my JavaScript was mostly copy-paste. Here, I had to understand event listeners, DOM manipulation, and how to make interactive elements that actually respond to user actions. The contact form was particularly tricky - getting form validation and submission working properly.

Professional Presentation

This wasn't a university assignment anymore. I needed to create something that potential employers would actually want to look at. Learning about color schemes, typography, spacing, and overall design principles became crucial.

What This Taught Me

  • Foundation Matters: Understanding the basics of how web technologies work together is more important than fancy features
  • User Experience: Building something that works well for visitors is different from building something that just works
  • Professional Standards: There's a big difference between student projects and professional work
  • Problem Solving: Debugging broken links and JavaScript errors teaches you more than following tutorials
  • Documentation: Keeping track of what you've built and how it works is essential

Keep in Touch Chat: Real-time Communication with AI

July 29, 2025 7 min read

Project Overview

Keep in Touch Chat is a real-time chat application I built to learn backend development and AI integration. It's a messaging platform where users can chat with each other, and there's an AI assistant that can help with conversations. This project taught me about databases, user authentication, and how to integrate external APIs.

Technical Stack

  • Backend: Python with Flask framework
  • Database: SQLite for storing users and messages
  • Authentication: Session-based login system
  • AI Integration: Google Gemini API for chat assistance
  • Frontend: HTML, CSS, JavaScript for the chat interface
  • Real-time Updates: JavaScript intervals for live message updates

Features I Built

  • User Registration & Login: Secure account creation and authentication
  • Real-time Chat: Messages update automatically without page refresh
  • AI Chat Assistant: Gemini AI integration for helpful responses
  • Message History: All conversations are saved and displayed
  • User Management: Profile system and user identification
  • Responsive Design: Works on desktop and mobile devices

Biggest Challenges

Understanding Backend Architecture

Coming from frontend-only projects, learning how servers work was completely new. I had to understand routes, request/response cycles, and how data flows between frontend and backend. Flask seemed overwhelming at first, but breaking it down into smaller pieces helped.

Database Design and Queries

Designing the database structure for users and messages was tricky. I had to think about relationships between tables and how to efficiently store and retrieve chat data. Writing SQL queries that actually worked took a lot of trial and error.

Real-time Updates Without WebSockets

I wanted real-time chat but didn't know about WebSockets yet. I used JavaScript intervals to poll the server for new messages every few seconds. It's not the most elegant solution, but it worked and taught me about client-server communication.

AI API Integration

Integrating Google's Gemini API was my first time working with external APIs. Understanding API keys, request formatting, and error handling was a steep learning curve. The documentation helped, but I still had to debug a lot of connection issues.

What I Learned

  • Backend Development: Building servers and handling requests is completely different from frontend work
  • Database Management: Proper data modeling is crucial for application functionality
  • API Integration: Working with external services requires understanding authentication and data formats
  • Security Basics: User authentication and session management are more complex than they appear
  • Real-time Systems: There are many ways to achieve real-time updates, each with trade-offs
  • Error Handling: Backend errors need to be handled gracefully and communicated to users

Trading Platform: Full-Stack Development with FastAPI

September 13, 2025 8 min read

Project Overview

The Resource Exchange Simulator is a full-stack trading platform that simulates a fake stock exchange where users can trade 5 different types of resources (Energy Units, Digital Tokens, Crypto Crystals, Bio Materials, and Rare Metals) with a starting balance of £10,000. This project was built to practice full-stack development with a focus on backend architecture, database management, and real-time updates.

Technical Stack

  • Backend: Python + FastAPI, SQLAlchemy ORM, SQLite database
  • Authentication: JWT tokens with bcrypt password hashing
  • Frontend: HTML5, CSS3 (Grid/Flexbox), JavaScript ES6+
  • Data Visualization: Chart.js for portfolio analytics and pie charts
  • Data Validation: Pydantic schemas for request/response validation
  • Real-time Updates: Server-side price generation every 2 seconds

Features Implemented

  • User Management: Registration, login, JWT-based authentication
  • Trading System: Buy/sell 5 different resource types with live pricing
  • Portfolio Tracking: Real-time portfolio valuation and P&L calculations
  • Transaction History: Complete audit trail of all trading activities
  • Live Price Updates: Simulated market prices that change every 2 seconds
  • Portfolio Analytics: Interactive charts showing portfolio breakdown and performance
  • Responsive Design: Mobile-friendly interface that works across all devices

Key Challenges & Solutions

Portfolio P&L Not Updating

The biggest challenge was ensuring real-time portfolio updates reflected current market prices. I initially forgot to call the `updatePositionsWithNewPrices()` function in the price update loop. The solution involved carefully sequencing the price updates with portfolio recalculations, ensuring that every price change immediately reflected in user portfolios and P&L calculations.

Database Session Management

SQLAlchemy session management proved tricky when updating user balances and positions. I was trying to update detached objects, which caused balance changes to not persist. The solution was switching to direct update queries instead of object manipulation, ensuring data consistency and proper transaction handling.

Frontend-Backend Synchronization

Ensuring the frontend displayed the most current data required careful orchestration of API calls. Initially, transactions weren't displaying because the frontend wasn't properly fetching the `/transactions` endpoint in the correct sequence. I implemented a proper fetch order that ensures portfolio data, transactions, and current prices are all synchronized.

Lessons Learned

  • Backend Architecture: FastAPI + SQLAlchemy provides powerful but complex backend capabilities that require careful session management
  • Real-time Systems: Real-time features need careful sequencing and error handling to maintain data consistency
  • Database Design: Proper database modeling is crucial for portfolio management and transaction tracking systems
  • Authentication Security: JWT + bcrypt provides robust security, but proper token management is essential
  • API Design: Well-structured APIs with proper validation make frontend integration much smoother
  • Error Handling: Database operations can break easily with a single wrong query - comprehensive error handling is vital
  • Testing Approach: Manual testing of edge cases revealed critical bugs that automated tests might miss
